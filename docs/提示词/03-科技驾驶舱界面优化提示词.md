# 科技驾驶舱界面优化提示词

## 使用场景

这份提示词用于让 AI 编程助手在现有 **DataSphereCenter / 数境中枢** 项目基础上，按照“科技驾驶舱风格”优化数据大屏界面。

目标是让页面从基础数据看板升级为更有科技感、更专业、更适合展示的数据大屏，并且最终自动启动项目、打开浏览器查看效果。

## 完整提示词

```text
你是一名资深数据大屏视觉设计师、前端架构师和 Vue 工程化专家。

当前项目是一个已经可以运行的数据可视化大屏项目。

项目英文名：DataSphereCenter
项目中文名：数境中枢

现有技术栈：

1. Vue 3
2. Vite
3. TypeScript
4. Pinia
5. Vue Router
6. ECharts
7. Axios
8. Vitest
9. Playwright
10. ESLint
11. Prettier
12. Stylelint

当前页面已经具备基础数据大屏结构，但整体视觉还比较单调。请你基于现有代码进行界面升级，不要推倒重来，不要破坏已有工程结构和测试体系。

本次优化目标：

将当前页面升级为“科技驾驶舱风格”的数据可视化大屏。

一、视觉风格目标

请将界面优化为以下风格：

1. 深色科技感背景
2. 数据驾驶舱视觉
3. 发光边框
4. 细线装饰
5. 面板角标
6. 顶部标题装饰线
7. 背景网格和数据流效果
8. 更有层次感的卡片和图表面板
9. 蓝色、青色、绿色、少量金色作为主辅助色
10. 适合 1920x1080 大屏展示

整体效果要像企业监控中心、运营驾驶舱、数据中枢展示屏，而不是普通后台管理页面，也不要做成营销落地页。

二、必须保留的项目结构

请继续遵守现有模块化结构：

src/
├─ app/
├─ assets/
├─ components/
├─ layouts/
├─ views/
├─ charts/
├─ services/
├─ mock/
├─ stores/
├─ types/
├─ utils/
├─ hooks/
├─ config/
├─ styles/
└─ tests/

要求：

1. 不要把大量代码堆到 App.vue。
2. 不要把所有图表写到一个巨大组件里。
3. 样式优先放到 styles 目录或组件对应样式中。
4. 图表组件继续放在 charts 目录。
5. 通用 UI 组件继续放在 components 目录。
6. 页面视图继续放在 views 目录。
7. 数据服务层、Mock/API 切换逻辑不要破坏。
8. 不要删除已有 README、LICENSE、测试配置和工程配置。

三、具体界面优化要求

请重点优化以下区域：

1. 顶部标题栏
   - 强化“数境中枢”主标题的视觉中心感。
   - 增加左右延展的科技装饰线。
   - 增加轻微发光效果，但不要刺眼。
   - 当前时间和系统状态要更像监控中心信息模块。
   - 系统状态可以显示 Online、运行正常、数据同步中等状态。

2. 背景层
   - 增加深色科技背景。
   - 增加细网格纹理。
   - 增加轻微数据流线、扫描线或缓慢移动的光带。
   - 背景动画要克制，不要影响图表阅读。
   - 不要使用大块纯色背景导致页面单调。

3. 指标卡片
   - 4 个核心指标卡要更有科技感。
   - 增加图标或抽象数据符号。
   - 增加趋势箭头或趋势状态。
   - 增加卡片底部光线、角标或渐变进度条。
   - 数字要突出，但不要溢出。
   - 卡片 hover 可以有轻微上浮或高亮。

4. 图表面板
   - 每个图表面板增加统一的驾驶舱面板样式。
   - 增加边框角标、顶部细线、标题英文副标题。
   - 图表色彩要统一，避免杂乱。
   - ECharts tooltip、legend、axis 样式要和整体风格一致。
   - 折线图要更有能量流动感。
   - 饼图或环形图要有发光色系。
   - 柱状图要有渐变柱体。

5. 实时告警列表
   - 优化为事件流样式。
   - 高、中、低等级用不同颜色区分。
   - 增加状态标签。
   - 列表项要更紧凑但清晰。
   - 可以增加轻微滚动或呼吸高亮效果，但不要过度。

6. 资源使用率区域
   - 优化为驾驶舱仪表模块。
   - 环形进度要更清晰。
   - 色彩与整体大屏统一。
   - CPU、内存、存储等信息要有层级。

四、交互与动效要求

请添加适度动效：

1. 页面加载时模块淡入。
2. 指标卡片轻微上浮。
3. 面板边框可以有轻微流光。
4. 背景可以有缓慢数据流动画。
5. 动画必须克制，不能影响性能和阅读。
6. 不要使用复杂 3D，不要引入 Three.js。
7. 不要加入过多闪烁效果。

五、响应式要求

请保证页面在以下场景中可用：

1. 1920x1080 大屏效果最佳。
2. 普通笔记本浏览器窗口不出现严重错位。
3. 小屏幕下模块可以自然换行。
4. 文字不能重叠。
5. 指标数字不能溢出卡片。
6. 图表容器尺寸稳定，不要因为加载或 hover 导致布局抖动。

六、代码质量要求

请在优化过程中保持代码质量：

1. 使用 TypeScript。
2. 保持组件职责单一。
3. 不要复制大量重复样式。
4. 尽量复用已有组件。
5. 如果需要新增装饰组件，可以放到 components 目录。
6. 如果需要新增公共样式变量，请放到 styles 目录。
7. 不要引入不必要的大型依赖。
8. 不要破坏 Mock/API 切换逻辑。
9. 不要破坏测试命令。

七、建议实现方向

你可以优先从以下文件入手：

1. src/styles/index.css
2. src/layouts/DashboardLayout.vue
3. src/components/MetricCard.vue
4. src/components/AlertList.vue
5. src/components/ResourceOverview.vue
6. src/charts/BusinessTrendChart.vue
7. src/charts/CategoryPieChart.vue
8. src/charts/RegionRankChart.vue
9. src/views/DashboardView.vue

如果现有文件结构已经不同，请先阅读项目结构，再按照实际情况修改。

八、验收要求

完成优化后，请执行以下检查：

1. npm run type-check
2. npm run lint
3. npm run test:unit
4. npm run build
5. 如项目已有 Playwright 配置，请执行 npm run test:e2e

如果检查失败，请自行修复，直到通过。

九、启动和浏览器查看要求

检查通过后，请启动本地开发服务器：

npm run dev

然后自动打开浏览器访问项目首页。

默认访问地址通常是：

http://127.0.0.1:5173

如果 5173 端口被占用，请使用 Vite 输出的实际端口，并告诉我最终访问地址。

浏览器中最终应该看到：

1. 标题“数境中枢”
2. 英文名“DataSphereCenter”
3. 科技驾驶舱风格背景
4. 更有层次的指标卡片
5. 更美观的图表面板
6. 更专业的实时告警和资源使用率模块

十、最终回复要求

完成后请告诉我：

1. 本次优化了哪些界面区域。
2. 修改了哪些核心文件。
3. 本地访问地址。
4. 执行并通过了哪些检查。
5. 是否存在非阻塞提示。
6. 后续还能继续优化哪些方向。
```
