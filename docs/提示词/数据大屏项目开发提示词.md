# 数据大屏项目开发提示词

## 使用场景

这份提示词用于让 AI 编程助手在现有仓库中，按照规范完整搭建一个可运行、可维护、模块化的数据可视化大屏前端项目。

项目英文名：DataSphereCenter

项目中文名：数境中枢

## 完整提示词

```text
你是一个资深数据大屏前端架构师和工程化专家，请直接在当前项目目录中执行开发工作，不要只输出方案。

我要开发一个数据可视化大屏项目。

项目英文名：DataSphereCenter
项目中文名：数境中枢

请基于 Vue3 + Vite + TypeScript 技术方案，完整搭建一个可运行、可维护、模块化的数据大屏前端项目。项目完成后，请启动本地开发服务器，并自动打开浏览器，让我看到一个漂亮、专业、科技感的数据可视化大屏页面。

一、技术架构要求

请使用以下技术栈：

1. Vue 3
2. Vite
3. TypeScript
4. Pinia
5. Vue Router
6. ECharts
7. Axios
8. Vitest
9. Playwright
10. ESLint
11. Prettier
12. Stylelint

项目当前阶段使用纯前端 Mock 数据，后期必须支持真实 API 切换。

二、核心目标

请完成以下事情：

1. 初始化 Vue3 + Vite + TypeScript 项目。
2. 建立规范的模块化目录结构。
3. 配置路由、状态管理、图表能力、请求工具和 Mock 数据层。
4. 实现 Mock/API 可切换的数据服务层。
5. 实现一个完整的数据可视化大屏首页。
6. 配置测试系统，包括单元测试和端到端测试。
7. 配置日志系统。
8. 配置代码质量系统。
9. 更新 README 文档。
10. 安装依赖并完成验收检查。
11. 启动本地开发服务器。
12. 自动打开浏览器展示最终页面。

三、目录结构要求

请按照以下结构组织代码，禁止把所有代码堆在一个文件里：

src/
├─ app/                 # 应用初始化、插件注册、路由配置
├─ assets/              # 图片、字体等静态资源
├─ components/          # 通用组件
├─ layouts/             # 大屏布局组件
├─ views/               # 页面视图
├─ charts/              # ECharts 图表组件
├─ services/            # 数据服务层
├─ mock/                # Mock 数据
├─ stores/              # Pinia 状态管理
├─ types/               # TypeScript 类型定义
├─ utils/               # 工具函数和日志工具
├─ hooks/               # Vue 组合式函数
├─ config/              # 全局配置
├─ styles/              # 全局样式、变量、重置样式
└─ tests/               # 测试代码

四、页面效果要求

请实现一个漂亮、现代、专业的数据大屏首页。

页面主题：数境中枢

页面必须包含：

1. 顶部标题栏
   - 中文标题：数境中枢
   - 英文标题：DataSphereCenter
   - 当前日期时间
   - 系统运行状态

2. 核心指标区
   至少包含 4 个指标卡片，例如：
   - 今日访问量
   - 活跃用户数
   - 数据接入量
   - 告警数量

3. 图表区
   至少包含以下模块：
   - 业务趋势折线图
   - 分类占比饼图或环形图
   - 区域排行柱状图
   - 实时告警列表
   - 系统资源使用率仪表盘、环形图或进度环

4. 视觉风格
   - 科技感
   - 深色背景
   - 蓝色、青色、绿色作为辅助色
   - 适合 1920x1080 大屏展示
   - 需要响应式适配常见浏览器窗口
   - 不要做成普通后台管理页面
   - 不要做成营销落地页
   - 不要使用大段说明文字占据页面

五、Mock/API 切换要求

必须封装统一的数据访问层。

页面组件不能直接 import Mock 数据。

请设计以下数据流：

页面组件
 -> services
 -> dataProvider
 -> mockProvider 或 apiProvider

通过环境变量切换数据来源：

VITE_DATA_SOURCE=mock
VITE_DATA_SOURCE=api

当前默认使用 mock。

要求：

1. mockProvider 返回本地模拟数据。
2. apiProvider 使用 axios 请求真实接口。
3. services 对页面暴露统一方法。
4. 页面组件不关心数据来自 Mock 还是真实 API。
5. 后期切换真实 API 时，页面组件不用修改。

六、日志系统要求

请封装 logger 工具。

要求：

1. 支持 debug、info、warn、error。
2. 开发环境允许输出到 console。
3. 生产环境默认关闭 debug。
4. 预留后期接入远程日志上报的位置。
5. 页面关键操作和数据加载失败需要使用 logger。

七、测试系统要求

请配置并编写基础测试。

要求：

1. 使用 Vitest 做单元测试。
2. 至少为工具函数或数据服务写一个单元测试。
3. 使用 Playwright 做端到端测试。
4. 至少测试首页能正常打开，并且能看到“数境中枢”标题。
5. 在 package.json 中提供测试命令：
   - test
   - test:unit
   - test:e2e

八、代码质量系统要求

请配置：

1. ESLint
2. Prettier
3. Stylelint

要求 package.json 中包含以下命令：

- lint
- lint:eslint
- lint:style
- format
- type-check
- build
- dev

九、工程规范要求

请遵循以下规范：

1. 使用 TypeScript。
2. 使用模块化开发。
3. 组件职责单一。
4. 页面只负责组合布局。
5. 图表组件放入 charts 目录。
6. 数据请求逻辑放入 services 目录。
7. Mock 数据放入 mock 目录。
8. 类型定义放入 types 目录。
9. 公共工具放入 utils 目录。
10. 全局样式放入 styles 目录。
11. 禁止把大量逻辑写在 App.vue 或 main.ts 中。
12. 禁止把所有图表写在一个巨大组件里。
13. 每个图表组件应该独立维护。
14. 代码要清晰，适合新手继续学习和维护。
15. 不要随意删除已有 README、LICENSE、Git 配置等项目基础文件。

十、README 文档要求

请更新 README.md，内容至少包含：

1. 项目介绍
2. 技术栈
3. 目录结构
4. 本地启动方式
5. 常用命令
6. Mock/API 数据切换方式
7. 测试说明
8. 代码质量说明
9. 开源协议说明

十一、验收要求

完成后请执行：

1. 安装依赖
2. 运行类型检查
3. 运行代码检查
4. 运行单元测试
5. 运行端到端测试
6. 构建项目
7. 启动开发服务器
8. 自动打开浏览器访问项目首页

如果某个步骤失败，请自行分析原因并修复，直到项目可以正常运行。

最终浏览器中应该能看到一个完整、美观、科技感的数据可视化大屏页面，标题为：

数境中枢
DataSphereCenter

十二、最终回复要求

完成后请告诉我：

1. 项目已完成哪些内容。
2. 本地访问地址。
3. 已执行并通过哪些检查。
4. 是否存在非阻塞提示，例如构建体积提示。
5. 后续建议我学习或开发的下一步。
```

